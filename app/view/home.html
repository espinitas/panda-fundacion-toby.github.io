Oracle APEX

Oracle APEX is an amazing tool which can be used to generate entire web applications using just an Oracle database. Let's say that you have a regular dabase table called Providers, well you can import that table into APEX and APEX will generate an entire, fully operational, and almost ready-to-production Web Application with the basic functionality that any profesional application must have:

  • Login Page
  • Signup Page
  • Users Administration
  • User's Roles
  • Security
  • Dynamic Reports
  • Charts
  • Rest Services
  • And more...
Isn't that cool?

I will use Oracle APEX as the datasource of my game, so here I plan to store the information related to the status of the game, the actions and the evaluation of the actions available to the players. Another excellent point to consider is that it is completely free to start working with Oracle APEX, you just have to go to APEX and follow the instructions to get your free workspace to start working with.

Oracle APEX has been aroung since 2004 (formerly know HTML DB), so now it is a very stable product and, according to what I've been told, it is a very popular product in Europe and Middle East.

One of its main characteristics is that APEX is highly integrated with the Oracle Database, so if you are using any version of Oracle DB you must be learning about APEX.

Google Application Engine for Java

Google Application Engine (GAP) for Java is the Google Cloud Solution for application development. With this tool I was able to run my Java code for my game and expose it as a Rest Service using Spring MVC.

I chose GAP because it is free to start develop, it is no necessary to introduce your credit card nor any other payment method and you can run any Servlet technologie into your container without any restrictions. So it is possible to deploy your EJB and, as in my case, any Spring application.
